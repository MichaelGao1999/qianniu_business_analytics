> 关联文件：`report-analyze.md`、`data/`、`reports/`

## 已完成

- [x] R-01: 设计四段式报告结构（整体/趋势/排行/总结）
- [x] R-02: 实现单店报告生成（整体指标 + 日度趋势 + 商品/渠道 TOP + 总结）
- [x] R-03: 实现多店合并报告生成（含店铺横向对比 TOP N）

## 待完成

### P1 — 高优先级
- [ ] R-05: **报告模板多样性（简洁版/高管版/运营版）**
  - 测试点：同一套数据生成三种不同风格的报告
  - 验收：通过 `--style` 参数或配置切换模板

- [ ] R-06: **Excel 分析容错增强（列名变化/空值处理）**
  - 测试点：Excel 列名与预期不一致时不崩溃
  - 验收：列名模糊匹配、空值填充默认值、缺失指标标注"无数据"

### P2 — 功能增强
- [ ] R-07: 增加数据可视化（生成简易图表/趋势图）
  - 测试点：报告中包含 Markdown 兼容的图表（如 ASCII 图或图片链接）
  - 验收：至少支持折线图（日度趋势）和柱状图（TOP N）

- [ ] R-08: 增加异常检测自动标注（环比下滑 >20% 自动标红）
  - 测试点：报告中有异常数据时自动提示
  - 验收：总结段增加"异常关注"小节

- [ ] R-09: 支持自定义报告标题和附加说明
  - 测试点：用户可指定报告标题和页脚备注
  - 验收：`--title` 和 `--note` 参数生效

- [ ] R-10: 报告历史版本管理（同一店铺+周期保留最近 N 份）
  - 测试点：重复生成时自动归档旧报告
  - 验收：`reports/` 下按日期子目录组织
