# 淘系店铺经营数据分析 — 需求提案

> 阶段一产出。本文档回答「要解决什么问题、为谁解决、做到什么程度算完成」。
> 后续设计（design.md）和任务拆分（tasks/）均以此为不可变更的边界。

---

## 1. 问题陈述

电商运营人员需要定期分析店铺经营数据（访客、成交、转化、商品表现等），但面临以下痛点：

1. **数据获取门槛高**：传统方式需要订购瓴羊·One 服务并申请 API 权限，流程繁琐
2. **分析效率低**：手动查看生意参谋后台，难以快速生成结构化的分析报告
3. **多店对比困难**：多家店铺的数据分散在不同后台，缺乏统一的横向对比工具
4. **报告分发不便**：分析结果需要手动整理成文档，再分发给团队

**核心诉求**：让运营人员通过最简路径（上传 Excel → 自动分析 → 获取报告）完成经营数据分析。

---

## 2. 目标用户

| 用户类型 | 特征 | 使用场景 |
|---------|------|---------|
| **电商运营人员** | 非技术背景，熟悉生意参谋后台操作 | 每周/每月定期分析店铺经营数据，向上汇报 |
| **AI Agent** | 自动化执行，需要明确的输入输出规范 | 用户通过对话触发分析，Agent 自动完成全流程 |

---

## 3. 功能范围（必须做）

### 3.1 核心功能

1. **Excel 数据读取**
   - 读取生意参谋导出的 `.xlsx` / `.xls` 文件
   - 自动识别 sheet 结构（整体指标 / 日度趋势 / TOP 排行）
   - 通过关键词模糊匹配核心指标列（访客数、支付金额、转化率、客单价等）

2. **数据分析**
   - 周期汇总核心指标
   - 按日展示关键指标走势
   - 商品/店铺维度 TOP 10 排行
   - 自动生成 3~6 条基于数据的量化洞察

3. **报告生成**
   - 统一四段式 Markdown 报告（整体指标 / 日度趋势 / TOP 排行 / 分析总结）
   - 支持单店报告和多店合并报告
   - 多店报告含横向对比（头部店铺 / 异常店铺 / Gap 分析）

4. **报告推送（可选）**
   - 飞书机器人推送（通过 `FEISHU_WEBHOOK` 环境变量配置）
   - 钉钉机器人推送（通过 `DINGTALK_WEBHOOK` 环境变量配置）

### 3.2 支持形态

| 形态 | 说明 |
|------|------|
| **单店分析** | 分析单个 Excel 文件，生成单店经营报告 |
| **多店合并** | 同时分析多个 Excel 文件，合并为一份含横向对比的报告 |

---

## 4. 功能边界（明确不做）

> 以下场景**必须明确拒绝**，不得擅自执行或伪造数据。

| 场景 | 原因 |
|------|------|
| **非淘系平台**（京东、抖音、拼多多、小红书等） | 数据口径不同（UV 定义、支付金额统计方式不同），强行合并会误导用户 |
| **全渠道 / 多渠道合并分析** | 跨平台数据对齐复杂度高，不在本技能范围内 |
| **广告投放分析**（万相台、直通车、钻展） | 属于独立专项，由 `qianniu_ad_analytics` 覆盖 |
| **VOC / 评论分析** | 属于独立专项，由 `jycm-voc-query` 覆盖 |
| **直播巡检** | 属于独立专项，由 `jycm-live-inspect` 覆盖 |
| **API 实时取数** | v5.0.0 起改为本地 Excel 驱动，降低使用门槛；旧版 API 脚本已归档 |
| **网页操作 / 模拟登录** | 禁止 Browser Use，避免触发安全风控 |

---

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| **数据真实性** | 严禁生成模拟数据、虚构数据、编造数据；Excel 读取失败必须如实告知 |
| **时区一致性** | 所有时间以北京时间（UTC+8）为准，统一使用 `T00:00:00+08:00` |
| **隐私安全** | 禁止在对话中输出明文 AK/SK；禁止将敏感信息提交到 Git |
| **容错性** | Excel 列名变化时尽量通过模糊匹配兼容；无法识别时列出检测到的列名引导用户 |
| **可扩展性** | 指标关键词映射表应易于扩展，便于支持新的生意参谋导出格式 |

---

## 6. 约束条件

| 约束 | 说明 |
|------|------|
| **数据源锚定** | 仅接受生意参谋导出的 Excel（`.xlsx` / `.xls`） |
| **渠道锁定** | `channelName` 固定为 `天猫淘宝`，`dataPlatform` 固定为 `生意参谋` |
| **技术栈** | Python 3.8+，依赖 pandas + openpyxl |
| **运行环境** | 本地 CLI / Agent 对话触发，无需服务器部署 |

---

## 7. 成功标准

> 项目做到什么程度可以认为「需求已满足」。

- [ ] 用户上传生意参谋 Excel 后，能自动识别指标并生成四段式 Markdown 报告
- [ ] 单店报告包含：整体指标、日度趋势、TOP 排行、分析总结
- [ ] 多店报告额外包含：店铺横向对比、头部/异常/Gap 分析
- [ ] 配置了 Webhook 后，能自动推送到飞书/钉钉
- [ ] 遇到非淘系诉求时，能明确告知用户不在范围内
- [ ] 所有报告结论基于真实 Excel 数据，无模拟/伪造数据
