# 淘系店铺经营数据分析 — 项目目标梳理

> 通过 grill-me 追问流程梳理，2026-06-25

---

## 1. 核心痛点

**自动化分析链路**：减少手动整理 Excel、写报告、发消息的重复劳动。

用户从生意参谋导出 Excel → 放到 `data/` 目录 → 运行命令 → 自动分析生成报告 → 推送到钉钉/飞书群。

---

## 2. 目标用户

**开源工具，谁都能用**：
- 个人运营
- 小团队（3-5 人）
- 部门级使用

设计要求：**易用性**和**可配置性**，不能假设使用者有技术背景。

---

## 3. 项目定位

**全能数据分析平台**（非轻量工具）：
- 支持更多数据源
- 支持更多分析维度
- 支持更丰富的报告模板
- 保持核心链路简单，通过插件/配置支持扩展

---

## 4. 技术方向

### 4.1 数据源：双模式支持

- **主模式**：Excel 驱动（用户手动导出 → 脚本分析）
- **辅模式**：API 驱动（自动从生意参谋拉数据，可选功能）

### 4.2 产物形态：Web 仪表盘 + CLI

**最终产物是 Web 应用**，CLI 保留作为底层能力和开发者工具。

| 层级 | 技术栈 | 职责 |
|------|--------|------|
| **前端** | Vue 3 + Element Plus | 数据可视化仪表盘、报告查看、配置管理 |
| **后端** | FastAPI (Python) | REST API，封装现有分析逻辑 |
| **分析引擎** | 现有 Python 脚本 | Excel 解析、报告生成、数据处理 |
| **CLI** | 现有脚本 | 开发者调试、自动化脚本、无 GUI 场景 |

## 6. 成功标准

当一个电商运营人员（非技术人员）能够：

1. 从生意参谋导出 Excel（或通过 API 自动拉取）
2. 放到指定目录（或配置好 API 凭证）
3. 运行一条命令
4. 在钉钉/飞书群收到一份**专业、多维度、多样化**的分析报告

并且这个过程**稳定可靠、文档清晰、易于维护、支持扩展**——项目就算完成。

### 具体完成条件

**核心功能**：
- [ ] 支持至少 5 种 dataType/dimension 组合（商品/品类/内容/客户/推荐）
- [ ] 支持至少 3 种报告模板（简洁版/标准版/高管版）
- [ ] Excel 解析容错：列名变化、空值处理、格式异常
- [ ] API 模式可选：Digest 认证、Token 刷新、错误重试

**Web 仪表盘**：
- [ ] 仪表盘首页：关键指标卡片、趋势图表、最近报告列表
- [ ] 报告查看：Markdown 渲染、导出 PDF/Excel
- [ ] 数据上传：拖拽上传 Excel、自动识别店铺、预览数据
- [ ] 配置管理：推送渠道配置、报告模板选择、定时任务设置

**工程质量**：
- [ ] 文档完整：README、使用指南、API 文档、贡献指南
- [ ] 测试覆盖：核心分析逻辑 80%+ 覆盖率
- [ ] 部署简单：Docker 一键启动 或 Streamlit Cloud 部署

## 7. 关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 数据源 | Excel 为主，API 为辅 | 稳定可靠 + 灵活可选 |
| 目标用户 | 开源，所有人可用 | 最大化影响力 |
| 功能定位 | 全能平台 | 满足深度分析需求 |
| 报告格式 | Markdown | 轻量、易推送、易扩展 |
| 推送渠道 | 钉钉 + 飞书 | 覆盖主流企业通讯工具 |
