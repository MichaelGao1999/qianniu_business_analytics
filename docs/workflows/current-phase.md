## 触发

## 标准动作序列

1. 读取 `status.md`
2. 检查关键字段是否为占位符（[填写]、[项目名]），如是则提示用户先填空
3. 按以下格式输出摘要：

```
【当前状态】
📌 所处阶段：阶段X — [阶段名]
✅ 已完成：[列出已完成的阶段 / 文件]
📋 待办：[从 status.md 待办章节取前 3 条]
🔴 阻塞项：[如有]
📁 关键入口文件：[列出 3-5 个核心文件]
🚀 建议下一步：[根据当前阶段给出一句建议]
```

4. 等待用户下一步指令
