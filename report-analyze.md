# 报告分析与推送规范

> 本文档定义报告格式、分析规则和钉钉推送规范。
> **范围限定：本技能仅输出淘系（天猫 / 淘宝 → 生意参谋）四段式报告（单店 / 多店通用），不产出跨平台（京东 / 抖音 / 拼多多等）或多平台合并报告。**
> **多店说明：支持一次性分析多家淘系店铺并合并为同一篇报告，店铺排行段用于横向对比；所有店铺必须同属淘系生意参谋。**

---

## 交付格式：Markdown（强制）

发给用户的报告与推送到钉钉的正文，**必须**均为 **Markdown**（与钉钉 `msgtype: markdown` 一致）。须使用：**分级标题**（`#` / `##`）、**有序/无序列表**、**Markdown 管道表格**、**粗体/行内代码**、**链接**。

**禁止**仅以无结构纯文字、仅截图、仅「见附件」作为唯一交付。

---

## 报告须回显给用户（强制）

生成报告后，**必须**在当前对话中向用户返回完整报告正文（Markdown）。

- **禁止**仅回复「已生成请查收」「已推送钉钉」等而无报告内容
- **禁止**仅以文件路径、OSS 链接、或钉钉推送代替对话中的全文交付
- 若正文过长可分条发送，但须保证用户能读到完整分析结论与数据呈现

---

## 淘系四段式报告（单店 / 多店通用，唯一报告形态）

本技能交付结果统一采用四段式报告。单店与多店使用同一结构，仅在「店铺排行 TOP N」和「分析总结」的展开粒度上有所差异：

1. **一、整体指标**（表：指标｜数值，列名以 Excel 为准）
   - 单店：展示该店铺区间汇总
   - 多店：展示所有被分析淘系店铺的合计 / 均值，并在下方补一张「分店铺汇总」小表
2. **二、日度趋势**（有日期列则按日表；无则说明）
   - 单店：按日绘制指标走势
   - 多店：按日×店铺呈现（可多张表或一张宽表），至少覆盖核心指标（如访客数、支付金额、转化率）
3. **三、店铺排行 TOP N**
   - 单店：若 Excel 内含店铺维度（如多绑定下的子维度）则排行；否则用商品 / 渠道 TOP 替代并说明
   - 多店：**必须**以店铺为主维度输出 TOP N 排行（默认 TOP 全量，≤10 家全部列出）
4. **四、分析总结**（3～6 条量化要点，基于上表）
   - 多店场景额外点出：头部店铺、尾部店铺、同比 / 环比 Gap 最大店铺、异常下滑店铺
5. **五、原始数据下载（必选，置于报告最末）**
   - 列出 `createAndDownload` 返回的完整下载地址（含 OSS URL）、文件名/任务标识、统计周期
   - 多店场景若为多次取数则逐一列出每家店铺的下载链接；若为一次 `shopIds` 多店请求则列出该次下载链接并注明覆盖店铺列表
   - **禁止**只写「已下载」而无可点击/可复制的链接

---

## 非淘系 / 全渠道场景处理（强约束）

当用户要求全渠道 / 多平台 / 京东 / 抖音 / 拼多多等非淘系报告时：

- **禁止**生成跨平台（淘系 + 京东 / 抖音 / 拼多多 …）合并报告
- **禁止**将淘系店铺报告伪装为「全渠道」报告
- **必须**告知用户：「本技能范围仅限淘系生意参谋，不执行跨平台合并分析；同为淘系的多家店铺可以合并为一篇」
- 如用户仅需淘系部分，确认后按淘系流程生成（单店或多店合并皆可）

## 飞书推送（可选·配置 Webhook 后自动启用）

仅在配置了 `FEISHU_WEBHOOK` 环境变量时执行推送。

### 配置方式

```bash
export FEISHU_WEBHOOK='https://open.feishu.cn/open-apis/bot/v2/hook/<your_token>'
```

### 推送规则

| 规则 | 说明 |
|------|------|
| **Markdown 格式** | 飞书 `text` 消息正文为 Markdown 源码 |
| **正文一致** | 推送正文 = 对话中已定稿的同一 Markdown（一源多用） |
| **标题必含「经营」** | 如「淘系店铺上周经营复盘」「经营周报」；不含时脚本自动加前缀 `经营·` |
| **超长截断** | 超过 18000 字符自动截断，对话中须说明截断处理 |
| **未配置时** | 跳过推送，提示用户可配置 `FEISHU_WEBHOOK` |
| **安全** | 勿在日志/截图中泄露 token |

### 推送脚本

```bash
# 从文件推送
python3 scripts/feishu_send_markdown.py --title '淘系店铺上周经营复盘' -f report.md

# 从 stdin 推送
cat report.md | python3 scripts/feishu_send_markdown.py --title '经营周报'
```

### 与对话内容一致（强制）

推送到钉钉的 Markdown 正文必须与对话中发给用户的报告正文**同源且一致**：

- 先定稿一份 `.md`，再原样用于：① 用户可见回复 ② `feishu_send_markdown.py` 的 `-f` 文件
- **禁止**对话里一版、钉钉里另写一版
- **禁止**对话用 Markdown、钉钉改用非 Markdown
- 若因长度限制截断钉钉正文，截断前须与对话报告逐字一致，并明确说明截断

## 通用执行流程

1. **解析用户请求**：识别店铺名称（**可为一家或多家淘系店铺**）、时间范围、数据类型/维度；若出现非淘系 / 跨平台诉求，先明确告知不在范围内；若涉及多家店铺且全部为淘系，则合并为一次多店分析
2. **取数**（按 [fetch-data.md](./fetch-data.md) 规则）：仅取一次（天猫淘宝 → 生意参谋）
3. **分析 Excel**：读取下载的 xlsx 文件
4. **生成报告**：四段式 + 文末下载链接，**在回复中向用户输出完整 Markdown 报告正文**
5. **钉钉推送**（可选）：若设置了 `DINGTALK_WEBHOOK`，用同一 Markdown 正文推送

**自动化脚本**：`python3 scripts/jycm_auto_report.py [--feishu]`

---

## 失败处理

- **openpyxl 未安装**：`pip install openpyxl`
- **用户要求跨平台 / 全渠道**：明确告知本技能范围仅限淘系生意参谋，不产出跨平台合并报告；同为淘系的多家店铺不受此限制，可合并为一篇多店报告
- **锚点路径选错但文件已下载**：不得将该文件当作有效交付，应重新按锚点取数
- **列名/口径不一致**：在同一报告内显式说明，避免强行等价

```bash
# 每周一上午 9 点自动推送淘系店铺上周经营报告
0 9 * * 1 cd /path/to/workspace && \
  python3 scripts/jycm_auto_report.py --days 7 --dingtalk
```

### 对话触发（推荐）

用户随时在对话中提出需求，Agent 实时执行完整流程。

### 混合模式

- 定时任务：每周一自动推送基础版
- 对话触发：用户随时请求自定义维度/时间
