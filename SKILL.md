# 淘系店铺经营数据分析（本地Excel驱动版）

> **本技能自包含全部能力**，无需外部API或子技能。所有分析、报告生成、推送的详细规范均在同目录子文档中。

## 强约束

> **数据分析是严肃场景，所有结论必须基于实际Excel中的数据，严禁任何形式的数据捏造。**

- **禁止**生成模拟数据、虚构数据、编造数据或使用占位数据充当真实结果。Excel 读取失败时必须如实告知，不得用假数据替代。
- 数据分析中涉及的所有时间均以**北京时间（UTC+8）**为准，不得使用其他时区的时间。
- **禁止**对千牛（qianniu）发起任何 Browser Use 操作（打开网页、模拟登录、操作页面等）。
- **禁止**将本技能误判为网页应用并尝试通过浏览器访问。
- **禁止**在对话中输出明文 AK/SK。

---

## 适用范围（强约束）

- **仅支持淘系店铺**：数据源锚定为生意参谋导出的 Excel（`.xlsx` / `.xls`）。
- **支持单店 / 多店两种形态**：同为淘系的一家或多家店铺可以在一次分析中合并为同一篇四段式报告（含「店铺排行 TOP N」横向对比）。
- **不支持全渠道 / 多平台合并**：若用户提出「全渠道」「各平台」「多渠道」「三平台」「京东」「抖音」「拼多多」等跨平台或非淘系诉求，**必须明确告知该场景不在本技能范围内**，不得擅自伪造其他平台数据，也不得将淘系结果伪装为全渠道结论。

## 触发条件

命中以下关键词时使用本技能（均为淘系场景）：
- 千牛、生意参谋、店铺经营、销售分析、流量分析、商品表现
- 「XX 店铺上周数据」「天猫店铺近 7 天销售」「天猫店铺上周成交」「天猫店铺销售」「淘宝店铺经营数据」

### 不触发

| 关键词 | 处置 | 原因 |
|--------|------|------|
| 全渠道、多渠道、三平台、京东、抖音、拼多多、小红书 | 明确告知用户「本技能仅支持淘系生意参谋」，不执行 | 非淘系场景 |
| VOC、原声、评论分析 | 转 `jycm-voc-query` / `jycm-voc-report` | VOC 专项 |
| 直播巡检 | 转 `jycm-live-inspect` | 直播专项 |
| 万相台 / 直通车 / 店铺推广 / 广告投放 | 转 `qianniu_ad_analytics` | 广告投放专项 |

## 执行流程（严格按序）

### 第一步：确认数据源

> **目标**：确认用户已准备好生意参谋导出的 Excel 文件。

用户说出需求后，Agent 应一次性询问：

1. **是否已从生意参谋导出 Excel？**
   - 生意参谋 → 对应模块 → 下载数据 → 保存为 `.xlsx`
2. **分析的时间范围？**（如：上周、近7天、本月）
3. **单店还是多店？**（多店需上传多个 Excel）

引导用户将 Excel 文件放入项目 `data/` 目录，或直接在对话中提供文件路径。

### 第二步：读取并分析 Excel

1. **扫描 `data/` 目录**下的 `.xlsx` / `.xls` 文件
2. **读取 Excel**：使用 `pandas` + `openpyxl` 读取所有 sheet
3. **识别结构**：
   - 含**日期列**的 sheet → 作为**日度趋势**数据源
   - 含**店铺/商品维度列**的 sheet → 作为**TOP排行**数据源
   - 其他数值 sheet → 作为**整体指标**数据源
4. **指标匹配**：通过关键词模糊匹配核心指标列（访客数、支付金额、转化率、客单价等）

> 若读取失败或列名无法识别，应列出检测到的 sheet 和列名，引导用户确认 Excel 格式。

### 第三步：生成报告与推送

按 `report-analyze.md` 生成报告并交付，**必须同时产出以下三项**：

1. **取数结果 Excel 文件**：原文件保留在 `data/` 目录
2. **Markdown 分析报告文件**：保存在 `reports/` 目录
3. **对话中输出报告**：在对话中向用户返回完整 Markdown 报告正文，并附上报告文件路径

报告格式要求：
- **淘系单店 / 多店通用** → 统一四段式报告（整体指标 / 日度趋势 / 店铺排行 TOP / 分析总结）+ 文末附原始数据文件路径
  - 单店：TOP 段位按时间或子维度（如渠道、商品）排列
  - 多店：TOP 段位以店铺为主维度做横向对比，总结段额外输出「头部店铺」「异常店铺」「Gap 最大店铺」等
- 若配置了 `FEISHU_WEBHOOK` → 自动推送飞书（同一 Markdown 源）

## 错误处理

| 问题 | 处理 |
|------|------|
| `data/` 目录无 Excel | 提示用户将生意参谋导出的 `.xlsx` 放入 `data/` 目录 |
| Excel 读取失败 | 列出文件信息，提示检查文件是否损坏或格式是否支持 |
| 列名无法识别 | 列出检测到的所有列名，提示用户核对是否为生意参谋标准导出格式 |
| 用户要求非淘系 / 全渠道数据 | 明确告知本技能范围仅限淘系生意参谋，不执行 |
| openpyxl / pandas 未安装 | 提示 `pip install pandas openpyxl` |
