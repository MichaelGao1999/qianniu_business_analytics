# Excel 上传规则

> 本文档定义如何将生意参谋导出的 Excel 文件用于分析。
> **范围限定：本技能仅处理淘系（天猫淘宝 → 生意参谋）导出的 Excel，不涉及京东 / 抖音 / 全渠道跨平台合并。**
> **多店支持：同为淘系的多家店铺可将多个 Excel 放入 `data/` 目录，合并为一份多店分析报告。**

---

## 必读文档

1. **报告规范**：同目录 `report-analyze.md`，定义四段式报告结构
2. **分析脚本**：`scripts/analyze_excel_report.py`，核心分析逻辑

---

## 文件上传规范

### 文件位置

用户需将生意参谋导出的 Excel 文件放入项目根目录的 `data/` 文件夹：

```bash
agentone_qianniu_business_analytics/
├── data/
│   ├── XX旗舰店_20260401_20260407.xlsx   # 单店文件
│   ├── YY官方店_20260401_20260407.xlsx   # 单店文件
│   └── ...
```

### 文件命名规范（推荐）

```
{店铺名称}_{开始日期}_{结束日期}.xlsx
```

示例：
- `XX旗舰店_20260401_20260407.xlsx`
- `多店汇总_20260401_20260407.xlsx`

> 命名不规范不影响分析，仅影响报告标题的自动推断。

## Excel 结构说明

生意参谋导出的 Excel 通常包含以下结构，分析脚本会自动识别：

### 类型一：含日期列的明细表（最常见）

| 日期 | 访客数 | 支付金额 | 支付转化率 | 支付买家数 | 客单价 | ... |
|------|--------|----------|------------|------------|--------|-----|
| 2026-04-01 | 12345 | ¥89,000 | 3.2% | 395 | ¥225 | ... |
| 2026-04-02 | 11200 | ¥76,500 | 3.0% | 340 | ¥225 | ... |

→ 被识别为 **日度趋势** 数据源

### 类型二：含商品/渠道维度的汇总表

| 商品名称 | 访客数 | 支付金额 | 支付转化率 | ... |
|----------|--------|----------|------------|-----|
| 商品A | 5000 | ¥35,000 | 4.1% | ... |
| 商品B | 3200 | ¥18,000 | 2.8% | ... |

→ 被识别为 **TOP排行** 数据源

### 类型三：单页汇总表

| 访客数 | 支付金额 | 支付转化率 | 支付买家数 | 客单价 | ... |
|--------|----------|------------|------------|--------|-----|
| 85600 | ¥623,000 | 3.1% | 2800 | ¥222 | ... |

→ 被识别为 **整体指标** 数据源

## 核心指标列名映射

分析脚本通过以下关键词自动匹配列名（大小写不敏感）：

| 标准指标 | 可识别的列名关键词 |
|----------|-------------------|
| 日期 | `日期`、`时间`、`date`、`day` |
| 访客数 | `访客数`、`浏览量`、`uv`、`访客`、`流量` |
| 支付金额 | `支付金额`、`成交金额`、`gmv`、`交易额`、`成交金额(元)` |
| 支付转化率 | `支付转化率`、`转化率`、`成交转化率`、`下单转化率` |
| 加购人数 | `加购人数`、`加入购物车人数`、`加购`、`购物车人数` |
| 支付买家数 | `支付买家数`、`支付人数`、`成交人数`、`买家数`、`付款人数` |
| 客单价 | `客单价`、`人均客单价`、`笔单价` |
| 店铺 | `店铺`、`店铺名称`、`店铺名`、`shop` |
| 商品 | `商品`、`商品名称`、`商品名`、`item` |

> 如果列名不在上述列表中，脚本会尝试智能匹配；匹配失败时会列出检测到的所有列名供排查。

## 多店合并分析

将多家淘系店铺的 Excel 同时放入 `data/` 目录：

```bash
python3 scripts/jycm_auto_report.py --shop "淘系多店汇总"
```

- 脚本会读取 `data/` 下所有 Excel
- 按店铺维度横向对比（需各 Excel 含店铺列或按文件名区分）
- 合并为同一篇四段式报告

---

## 失败与澄清

- **Excel 读取失败**：检查文件是否为标准 `.xlsx` / `.xls` 格式，是否被加密或损坏
- **列名无法识别**：脚本会输出检测到的所有 sheet 和列名，请核对是否为生意参谋标准导出
- **Sheet 结构异常**：生意参谋不同模块导出的 Excel 结构不同，若遇特殊情况可手动指定分析重点
- **数据为空**：确认 Excel 中确实有数据行（非仅表头）
